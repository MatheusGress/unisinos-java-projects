# 💻 Exercício 3 — Modelagem de Classes em Java (BlueJ)

## 📖 Descrição

Este projeto foi desenvolvido em **Java**, utilizando o **BlueJ**, com o objetivo de praticar a criação e modelagem de múltiplas classes aplicando conceitos fundamentais de **Programação Orientada a Objetos (POO)**.

O exercício consiste na criação de diferentes classes representando objetos do mundo real, cada uma com seus atributos, construtores e métodos de acesso.

---

## 🧱 Estrutura do Projeto

O projeto contém as seguintes classes:

- `Teclado`
- `Cor`
- `Computador`
- `Lampada`
- `Produto`

---

## 🎨 Classe `Cor`

Representa uma cor no padrão RGB.

**Atributos:**
- `r` — intensidade do vermelho (int)
- `g` — intensidade do verde (int)
- `b` — intensidade do azul (int)

**Implementações:**
- Construtor para inicialização dos atributos
- Métodos de acesso (getters e setters)

---

## 💻 Classe `Computador`

Representa um computador com suas características principais.

**Atributos:**
- marca (String)
- velocidade (double ou int)
- ano de fabricação (int)
- indica se o computador é novo (boolean)

**Implementações:**
- Construtor
- Métodos de acesso (getters e setters)

---

## 💡 Classe `Lampada`

Representa o estado de uma lâmpada.

**Atributo:**
- ligada (boolean)

**Implementações:**
- Construtor
- Métodos de acesso (getters e setters)

---

## 📦 Classe `Produto`

Representa um produto armazenado em estoque.

**Atributos:**
- nome (String)
- valor (double)
- descrição (String)
- quantidade em estoque (int)

**Implementações:**
- Construtor
- Métodos de acesso (getters e setters)

---

## ⌨️ Classe `Teclado`

Classe fornecida no projeto para possibilitar leitura de dados via teclado, auxiliando na interação com o usuário durante testes e execuções.

---

## 🧠 Conceitos Aplicados

- Programação Orientada a Objetos (POO)
- Modelagem de classes
- Encapsulamento
- Construtores
- Métodos de acesso (getters e setters)
- Tipos primitivos e objetos em Java

---

## 🚀 Como Executar

1. Abra o projeto no **BlueJ**.
2. Compile todas as classes.
3. Instancie os objetos desejados para testes diretamente pelo ambiente do BlueJ ou através de uma classe principal, se criada.

---

## 🎯 Objetivo Educacional

O exercício tem como finalidade reforçar a criação de classes independentes, organização de atributos e aplicação correta do encapsulamento em Java.
